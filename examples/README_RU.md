В этом каталоге содержатся готовые к эксплуатации примеры ботов, построенных на базе фреймворка G-MAN. Эти примеры демонстрируют использование системы модулей, торгового движка и пользовательской бизнес-логики.

## 🚀 Начало работы

Чтобы запустить любой из примеров, выполните следующие шаги:

1. **Настройте переменные окружения:**
   Ботам требуются ваши учетные данные Steam, установленные в переменной окружения.
   ```bash
   # Windows (PowerShell)
   $env:STEAM_USER="ваш_логин"
   $env:STEAM_PASS="ваш_пароль"

   # Linux/macOS
   export STEAM_USER="ваш_логин"
   export STEAM_PASS="ваш_пароль"
   ```

2. **Запустите пример:**
   ```bash
   go run ./examples/trading_bot
   ```

## 🏗 Общие паттерны

### 1. Цепочка Middleware
G-man использует паттерн "Onion" (луковая шелуха) для торговой логики. Каждое промежуточное ПО может либо:
- Вынести **Вердикт** (Принять/Отклонить/Контрпредложение) и остановить цепочку.
- Передать контекст **Следующему** middleware в цепочке.

### 2. Автоматизированный процессор
Вместо ручного принятия трейдов используйте `SetOfferHandler` со встроенным `engine.NewBotHandler`. Он обрабатывает:
- **Блокировку предметов (Item Locking):** Предотвращает продажу одного и того же предмета дважды.
- **Повторы (Retries):** Автоматически повторяет вызовы WebAPI с экспоненциальной задержкой.
- **Конкурентность:** Обрабатывает офферы последовательно, чтобы избежать состояния гонки в инвентаре.

### 3. Постоянное хранилище
Все примеры используют `storage.json` для сохранения:
- Токенов входа (позволяет избежать частых запросов 2FA).
- Секретов Steam Guard.
- Идентификаторов оборудования для конкретной машины.

## 🛠 Продвинутые возможности TF2 (Пример TF2 Bot)

Бот для TF2 — самый продвинутый пример, требующий дополнительной настройки:
- **`trading_config.json`**: Генерируется при первом запуске. Позволяет устанавливать лимиты запасов и цены для каждого SKU.
- **API Ключи**: Требуются `BPTF_API_KEY` и `BPTF_USER_TOKEN` для проверки дубликатов и мониторинга репутации.
- **PriceDB**: Использует единый фид цен, чтобы ваш бот всегда покупал дешево и продавал дорого.
